
# Projeto de Interface

Dentre as preocupações para a montagem da interface do sistema, será estabelecido foco em questões como agilidade, acessibilidade e usabilidade. Desta forma, o projeto tem uma identidade visual padronizada em todas as telas que são projetadas para funcionamento em desktops.

## Diagrama de Fluxo

O diagrama apresentado na Figura 1 mostra o fluxo de interação do usuário pela navegação do sistema. Cada uma das telas deste fluxo é detalhada na seção de Wireframes que se segue. Para visualizar o Wireframe interativo, acesse : https://marvelapp.com/prototype/65hdfcd/screen/88873663

## Wireframes Interativos 

Conforme a navegação do projeto apresentado no item anterior, às telas do sistema são apresentadas em detalhes nos itens que se seguem e apresentam uma estrutura comum. Nesta estrutura, existe um grande bloco, descrito a seguir.

## Cabeçalho

Cabeçalho: Local onde se dispõem elementos fixos de identidade (Logo) e navegação principal do site, como barra de pesquisa destinada a procura especifica de profissionais e serviços (Menu de aplicação).

## Tela: Home page 

Tela de Home page é composta por dois blocos, sendo eles;<br>
Componentes de apresentação de forma simplista e intuitiva sobre a ideia contida no projeto.<br>
Componente em formato carrossel apresentada como portifólio de diversos profissionais e serviços já vinculados a plataforma.

![image](https://user-images.githubusercontent.com/101372729/193457060-03d844bc-f804-43ec-b3b9-615934956bb5.png)

## Tela: Busca de Serviços

Tela responsável pelo resultado de busca feito através da barra de pesquisa contida no cabeçalho, apresentando os profissionais em forma de colunas no bloco principal, tendo a possibilidade de filtros como preço e localidade.

![image](https://user-images.githubusercontent.com/101372729/193457429-de1faedb-1680-47f3-bb0e-68c94ccb8ab6.png)

**Pagina completa e interativa, acesse:** [Busca-Serviços](https://marvelapp.com/prototype/65hdfcd/screen/88892350)

## Tela: Cadastro de Usuário

Tela responsável por cadastro dê Anunciante é contratante, visto que há necessidade de vinculo a plataforma para dar seguimento a contrações de serviços e publicações de serviços é anúncios a plataforma, com isso há necessidade de complementação de informações como;<br>

- NOME <br>
- E-MAIL<br>
- SENHA<br>

## Tela: Login Usuário

Tela responsável pelo login de usuário já cadastrado na plataforma, necessitando o preenchimento dos campos de e-mail e senha, caso o usuário não possua vinculo a plataforma, há um campo “Cadastre -se” direcionando para tela cadastro de usuário.

![image](https://user-images.githubusercontent.com/101372729/193457634-3be2cf5d-916f-4958-9eb7-b1f8ab0b1960.png)

**Pagina completa e interativa, acesse:** [Login-Usuário](https://marvelapp.com/prototype/65hdfcd/screen/88874865)
